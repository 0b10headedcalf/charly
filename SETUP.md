# DigitalOcean setup (do this once, ~15 min)

The app has three tiers. It works at every tier, so do them in order and stop
when you run out of time:

| Tier | What you get | What you need |
|---|---|---|
| 0 | Demo mode: scripted Charli, template plans | nothing |
| 1 | Live Charli + live plans + live classification | 1 model access key |
| 2 | Console **agents** + **knowledge base RAG** with citations (the full "Gradient AI" judging story) | 2 agents in the console |

## Tier 1 â€” model access key (5 min)

1. [cloud.digitalocean.com](https://cloud.digitalocean.com) â†’ left menu **INFERENCE** â†’ **Manage** â†’ **Create model access key**. Copy the secret immediately (shown once).
2. `cp .env.example .env.local`, paste it as `DIGITAL_OCEAN_MODEL_ACCESS_KEY`.
3. Restart `npm run dev`. Chat + plans + classification are now live models
   (`anthropic-claude-4.6-sonnet` and `llama3.3-70b-instruct` via
   `https://inference.do-ai.run/v1/`).

Sanity check:

```bash
curl -s -X POST https://inference.do-ai.run/v1/chat/completions \
  -H "Authorization: Bearer $DIGITAL_OCEAN_MODEL_ACCESS_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "llama3.3-70b-instruct", "messages": [{"role":"user","content":"hi"}], "max_completion_tokens": 10}'
```

## Tier 2 â€” console agents + knowledge base (10 min)

Console â†’ **INFERENCE** â†’ **Agent Platform** (Workspaces).

### Charli agent
1. Create agent, name `charli`. Model: Claude Sonnet (or any chat model).
2. Instructions: paste the ENTIRE `CHARLI_SYSTEM_PROMPT` string from
   `src/lib/prompts.ts` (everything between the backticks â€” includes the
   MATCH_READY protocol line, which the app parses).
3. After it deploys, open the agent â†’ **Endpoint** â†’ create an **agent access key**.
4. In `.env.local`: `CHARLI_ENDPOINT=https://<id>.agents.do-ai.run` and `CHARLI_KEY=<key>`.

### Knowledge base + Planner agent
1. Agent Platform â†’ **Knowledge bases** â†’ create one named `charly-orgs` â†’ add
   data source â†’ **file upload** â†’ upload `data/orgs.json` from this repo. Wait
   for indexing to finish.
2. Create agent `planner`. Instructions: paste `PLANNER_SYSTEM_PROMPT` from
   `src/lib/prompts.ts`. Attach the `charly-orgs` knowledge base (Resources tab).
3. Create its access key; set `PLANNER_ENDPOINT` + `PLANNER_KEY` in `.env.local`.
4. Restart dev server. Group plan pages will now show
   "Generated by DigitalOcean agent + knowledge base" plus RAG source citations.
